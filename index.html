가상 메모리 (Virtual Memory)

EBS 수능특강 독서 041쪽 | 개념 학습 03-③

컴퓨터가 실행에 필요한 프로그램의 일부만을 주기억 장치에 올려놓고, 나머지는 보조 기억 장치에 두는 메모리 관리 기법. 프로그램을 같은 크기의 페이지로 나누어 필요한 페이지만 올리는 페이징(Paging) 방식을 주로 사용한다.

보조 기억 장치

주기억 장치 vs 보조 기억 장치

주기억 장치(RAM)는 용량이 작지만 속도가 빠르고, 보조 기억 장치(HDD/SSD)는 용량이 크고 영구 저장 가능. 가상 메모리는 보조 기억 장치를 메모리처럼 활용한다.

Paging
페이징

페이지 & 페이지 프레임

프로그램을 같은 크기의 단위인 페이지로 분할하고, 주기억 장치의 수용 공간을 페이지 프레임이라 한다. 페이지 테이블이 가상 주소↔실제 주소를 매핑한다.

Page Fault
페이지 부재

페이지 부재 처리 과정

실행에 필요한 페이지가 주기억 장치에 없는 상황. 보조 기억 장치에서 해당 페이지를 복사하여 페이지 프레임에 올린 후 실행을 재개한다.

Page Replacement
페이지 교체

페이지 프레임이 가득 찬 경우

기존 페이지를 내보내고 새 페이지를 올려야 할 때 발생. 내보낼 페이지 내용이 수정되었으면 먼저 보조 기억 장치에 저장 후 교체한다.

페이지 교체 알고리즘 비교
방식 정의 장점 단점
FIFO 가장 먼저 들어온 페이지부터 교체 (선입선출) 구현 단순 자주 쓰는 페이지도 삭제될 수 있음
LRU 가장 오랫동안 사용되지 않은 페이지 교체 시간 지역성 활용, 효율적 오랜 미사용이 앞으로 안 쓰인다는 보장 없음
LFU 사용 빈도가 가장 낮은 페이지 교체 실제 참조 빈도 반영 최소 빈도 2개 이상 시 추가 규칙 필요
📌 수능 출제 예시 (프레임 3개: 0,1,2 / 사용 순서: A→B→C→B→A→D)
페이지 프레임에 A, B, C가 있을 때 D가 필요한 상황:
· FIFO: 가장 먼저 들어온 A 교체 → A가 선출 대상
· LRU: 최근에 가장 안 쓴 C 교체 (최근 B→A→D 순 참조)
· LFU: 사용 빈도가 1회인 C 교체 (A·B는 2회 이상)

🖥 MMU (메모리 관리 장치)

가상 메모리 주소를 실제 메모리 주소로 변환하는 하드웨어. 페이징 등 메모리를 일정 단위로 관리한다.

🔄 스와핑 (Swapping)

프로세스 전체를 주기억 장치와 보조 기억 장치 사이에 이동시키는 기법. 페이징과 구분할 것.

📍 지역성 원리 (Locality)

최근 참조된 데이터나 인접 데이터를 다시 참조할 가능성이 높다는 원리. LRU의 이론적 근거.

⚠️ 스래싱 (Thrashing)

페이지 교체가 지나치게 빈번하여 CPU 이용률이 오히려 급격히 낮아지는 현상.

📋 페이지 테이블

프로세스마다 존재. 가상 페이지 번호 → 물리 프레임 시작 주소 매핑 정보 기록.

💡 운영 체제의 역할

페이지 테이블 관리, 페이지 부재 처리, 페이지 교체 결정 등 가상 메모리 전반을 총괄한다.

CPU와 GPU의 차이

CPU는 소수의 고성능 코어로 복잡한 제어·순차 작업에 최적화. GPU는 수백~수천 개의 단순 코어로 동일 연산을 대량 병렬 처리하는 구조.

🧠 NPU (신경망 처리 장치)

AI 연산에 특화된 장치. GPU보다 전력 효율이 높고 AI 추론에 최적화. 스마트폰 등에 내장.

⚡ SIMD 구조

"Single Instruction, Multiple Data" — 하나의 명령어로 여러 데이터를 동시에 처리. GPU 병렬 처리의 핵심 원리.

🔢 딥러닝과 행렬 연산

AI 학습의 핵심인 행렬 곱셈·역전파 연산은 동일한 연산의 반복 → GPU가 최적화되어 있어 딥러닝에 활용.

🌡 열 병목과 전력 효율

GPU는 코어 수가 많아 발열·전력 소모가 큼. 이를 해결하기 위한 칩 설계와 냉각 기술이 중요한 연구 분야.

🔗 CPU-GPU 협력 구조

CPU가 전체 흐름 제어, GPU가 반복 연산 담당. 연산 흐름을 유기적으로 조율하는 소프트웨어 설계가 핵심.

📡 CUDA / OpenCL

GPU 병렬 연산을 프로그래밍하기 위한 플랫폼. GPU의 범용 계산(GPGPU)을 가능하게 하는 소프트웨어 환경.

OX 퀴즈 — 가상 메모리

각 문항을 읽고 O(맞음) 또는 X(틀림)를 선택하세요

💾 가상 메모리 핵심 OX 퀴즈

1 주기억 장치는 보조 기억 장치보다 저장 용량이 크다.
❌ 틀림 (X) — 주기억 장치(RAM)는 속도가 빠르지만 용량이 작고, 보조 기억 장치(HDD/SSD)가 용량이 크다.
2 가상 메모리는 보조 기억 장치의 일부를 메모리처럼 사용하는 기법이다.
✅ 맞음 (O) — 가상 메모리는 보조 기억 장치의 일부를 '메모리'처럼 쓴다는 의미에서 '가상 메모리'라 불린다.
3 페이징에서 페이지와 페이지 프레임의 크기는 서로 다를 수 있다.
❌ 틀림 (X) — 페이지와 페이지 프레임의 크기는 같다. 페이지 단위로 프레임에 들어가기 때문이다.
4 페이지 교체 시 교체할 페이지의 내용이 수정되었으면 보조 기억 장치에 먼저 저장한 후 제거한다.
✅ 맞음 (O) — 내용이 수정된 페이지는 보조 기억 장치에 수정 사항을 기록한 후 제거해야 데이터 손실을 막을 수 있다.
5 FIFO 방식은 미래에 자주 사용될 페이지를 고려하여 교체 대상을 선정한다.
❌ 틀림 (X) — FIFO는 가장 먼저 들어온 순서대로 교체하며, 미래의 참조 가능성을 전혀 고려하지 않는다. 그래서 비효율적인 상황이 발생할 수 있다.
6 LRU 알고리즘은 '시간 지역성' 원리를 근거로 한다.
✅ 맞음 (O) — LRU는 최근에 참조된 페이지가 미래에도 다시 참조될 가능성이 높다는 시간 지역성에 근거한다.
7 LFU 방식에서 최소 빈도 페이지가 2개 이상이면 추가 규칙이 필요하다.
✅ 맞음 (O) — LFU에서 최소 빈도가 같은 페이지가 2개 이상이면, 예를 들어 FIFO나 LRU 방식을 추가 적용하는 등 별도 규칙이 필요하다.
8 가상 메모리 주소와 실제 메모리 주소는 항상 일치한다.
❌ 틀림 (X) — 가상 메모리 주소와 실제 메모리 주소는 일치하지 않는다. MMU(메모리 관리 장치)가 이를 변환해준다.

OX 퀴즈 — GPU

각 문항을 읽고 O(맞음) 또는 X(틀림)를 선택하세요

🖥️ GPU 핵심 OX 퀴즈

1 GPU는 소수의 고성능 코어로 복잡한 제어 흐름을 처리하는 데 최적화되어 있다.
❌ 틀림 (X) — 이는 CPU의 특징이다. GPU는 수백~수천 개의 단순 코어로 동일 연산을 병렬 처리하는 데 최적화되어 있다.
2 하나의 스레드는 하나의 명령어를 실행하는 최소 연산 단위다.
✅ 맞음 (O) — 스레드는 연산을 수행하는 최소 단위로, 하나의 스레드는 하나의 명령어를 실행한다.
3 GPU의 전역 메모리는 접근 속도가 빠르지만 스레드 블록 내부에서만 공유된다.
❌ 틀림 (X) — 전역 메모리는 모든 스레드가 접근할 수 있지만 접근 속도가 느리다. 스레드 블록 내 공유는 '공유 메모리'의 특징이다.
4 GPU는 분기문이나 조건 판단이 자주 등장하는 연산에서 병렬 처리의 효율이 낮아질 수 있다.
✅ 맞음 (O) — 분기문이 삽입되면 스레드마다 실행할 작업이 달라져, 일부 스레드가 대기하게 되어 병렬성이 손실된다.
5 레지스터는 GPU 메모리 중 접근 속도가 가장 빠르다.
✅ 맞음 (O) — 레지스터는 용량은 작지만 접근 속도가 매우 빠르다. GPU 메모리는 속도 순으로 레지스터 > 공유 메모리 > 전역 메모리다.
6 스케줄러는 GPU에 전달된 명령어에 따라 스레드 블록을 각 연산 유닛에 분배하고 병렬 처리를 제어한다.
✅ 맞음 (O) — 스케줄러는 명령어를 해석하여 스레드 블록을 연산 유닛에 배정하고 병렬 처리가 이루어지도록 총괄 제어한다.
7 현대 컴퓨팅 시스템은 CPU와 GPU 중 하나만 선택하여 운용한다.
❌ 틀림 (X) — 현대 컴퓨팅 시스템은 CPU와 GPU의 장점을 모두 살리기 위해 두 장치를 함께 운용하는 구조로 발전하고 있다.
8 AI 학습에서 행렬 곱셈이나 역전파 연산은 동일 연산의 반복이므로 GPU 활용에 적합하다.
✅ 맞음 (O) — AI 학습의 핵심인 행렬 연산·역전파는 동일 연산의 대규모 반복으로, GPU의 병렬 처리 구조에 매우 적합하다.
